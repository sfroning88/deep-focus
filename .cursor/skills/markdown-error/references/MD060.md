# MD060 - Table column style

**Source:** [markdownlint/doc/Rules.md](https://github.com/DavidAnson/markdownlint/blob/main/doc/Rules.md) (DavidAnson/markdownlint)

## Rule Summary

**Tags:** `table`  
**Aliases:** `table-column-style`

This rule is triggered when the column separator pipe characters (`|`) of a [GitHub Flavored Markdown table](https://docs.github.com/en/get-started/writing-on-github/working-with-advanced-formatting/organizing-information-with-tables) are used inconsistently.

## Parameters

- `aligned_delimiter`: Aligned delimiter columns (`boolean`, default `false`)
- `style`: Table column style (`string`, default `any`, values `aligned` / `any` / `compact` / `tight`)

## Supported Styles

### Style `aligned`

Ensures pipe characters are vertically aligned. Requires padding cell content so pipes line up.

```markdown
| Character | Meaning |
| --------- | ------- |
| Y         | Yes     |
| N         | No      |
```

**Note:** Pipe alignment is based on visual appearance, not character count. Emoji and CJK characters render at twice the width of Latin characters.

### Style `compact`

Avoids extra padding with a single space around cell content. **Recommended for tables with long content (e.g., image URLs).**

```markdown
| Character | Meaning |
| --------- | ------- |
| Y         | Yes     |
| N         | No      |
```

### Style `tight`

Uses no padding at all for cell content:

```markdown
| Character | Meaning |
| --------- | ------- |
| Y         | Yes     |
| N         | No      |
```

### Style `any` (default)

Each table is analyzed to see if it satisfies any supported style. If so, no violations are reported. If not, violations are reported for whichever style would produce the fewest issues.

## Why Compact Works

When the rule uses `any` (default), a table matching **any** of the four styles passes. Compact is the easiest to satisfy for tables with variable-length content (images, URLs) because it does not require pipe alignment.
